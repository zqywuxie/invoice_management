# 发票管理系统启动说明

## 依赖安装

```powershell
pip install -r requirements.txt
```

## 配置文件约定

项目现在支持两套运行配置：

- 本地开发：`SQLite`
- 远程服务器：`PostgreSQL`

推荐做法：

```powershell
Copy-Item .env.local.example .env.local
Copy-Item .env.server.example .env.server
```

应用启动时会按下面顺序自动寻找配置文件：

1. `ENV_FILE` 指定的文件
2. `.env.local`
3. `.env.server`
4. `.env`

如果当前进程里已经设置了同名环境变量，则进程环境优先，不会被文件覆盖。

## 本地开发：继续使用 SQLite

本地配置文件示例：

```env
DATABASE_URL=sqlite:///data/invoices.db
APP_PORT=5000
```

启动：

```powershell
python invoice_web/run.py
```

默认访问地址：

```text
http://127.0.0.1:5000
```

## 远程服务器：使用 PostgreSQL

服务器配置文件示例：

```env
DATABASE_URL=postgresql://invoice_user:your_password@127.0.0.1:5432/invoice_db
APP_PORT=5001
DB_POOL_MIN=2
DB_POOL_MAX=20
```

如果你是直接运行 Python 进程，启动前指定配置文件：

```powershell
$env:ENV_FILE=".env.server"
python invoice_web/run.py --host 0.0.0.0
```

如果你使用 Docker Compose，推荐直接指定服务器配置文件：

```powershell
docker compose --env-file .env.server up -d --build
```

## 数据迁移

把本地 SQLite 数据迁移到 PostgreSQL：

```powershell
$env:ENV_FILE=".env.server"
python scripts/migrate_sqlite_to_postgres.py `
  --sqlite-path data/invoices.db
```

如果已经在命令行传入 `--postgres-url`，则会优先使用命令行参数。

## 运行时确认

启动日志里会打印两项关键信息：

- `Env file: ...`
- `Database backend: ...`

你可以用它确认当前实例到底加载了哪份配置，以及实际连接的是 `SQLite` 还是 `PostgreSQL`。
